# Launch plan

## 1. Make the repository look serious

- Add a clear README
- Add vision, roadmap, architecture, and contribution docs
- Add a simple CMake build entry point
- Add a GitHub Actions workflow

## 2. Publish a first milestone

Create a public milestone such as:

- Transparent window
- Rounded rectangle rendering
- FPS counter

## 3. Ask for collaborators in a focused way

Post a message that explains:

- what the project is
- what is already built
- what skills you want help with
- what the first good tasks are

## 4. Use GitHub issues as the onboarding path

Open issues such as:

- Build on Windows with Visual Studio
- Add a first shader
- Create a widget architecture proposal
- Improve documentation

## 5. Keep the project public and visible

Share one progress update per week with:

- screenshots
- a short technical note
- a clear next step
